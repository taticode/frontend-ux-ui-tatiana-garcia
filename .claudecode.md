# Critical Behavioral Rules for Claude Code

## 1. Token Efficiency & Communication
- **No Verbosity:** Do not explain why the code is better or what was changed unless explicitly asked. Provide only the concise final code.
- **Dry Run First:** Always propose a brief 3-bullet point summary of changes before writing to files. Wait for user approval.
- **Scope Control:** Only analyze and modify files directly related to the current task. Do not scan the entire project unnecessarily.

## 2. Technical Architecture
- **Icon Management:** Use a single TypeScript constant file (`src/app/shared/constants/icons.ts`) to store all SVG paths as a dictionary. 
- **No Over-Engineering:** Do not create new components for icons. Use `[attr.d]` to inject paths into lightweight SVGs.
- **DRY Logic:** Prioritize Angular's `@for` and configuration arrays over repetitive HTML structures.

## 3. Quality & Accessibility (a11y)
- **a11y Mandate:** Every interactive element MUST have an `aria-label`. 
- **Semantic Images:** Every `<img>` must have an `alt` attribute (descriptive if functional, empty `alt=""` if decorative).
- **Focus Management:** Ensure all refactored buttons maintain visible focus states.

## 4. Stability
- **Preserve Design:** Never change the visual layout, positions (top, left, absolute), or specific CSS styles unless instructed.
- **Comments:** Preserve existing code comments in their original language.
